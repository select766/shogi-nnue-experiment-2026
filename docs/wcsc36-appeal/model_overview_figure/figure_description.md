## 脚注

1. 「※ DNN推論は1手番1回のみ (ルート局面)。探索速度への影響は無視できる。」
2. 「※ 加重平均はエキスパート評価値のアンサンブルではなく、重みパラメータ空間での補間。」

---

## 意味的に重要な設計上のポイント

1. **NNUE Experts は静的なパラメータ集合**: 推論時には全エキスパートの重みセットが加重平均に渡される（動的ルーティングではない）
2. **パラメータ空間での補間**: エキスパートの「出力値」ではなく「重みパラメータ」を混合する
3. **DNN推論はルート局面のみ**: DNN左列は1手番に1回だけ実行。探索中の各局面にはHalfKP特徴量が直接やねうら王へ渡される
4. **やねうら王は学習対象外**: 探索エンジン自体は変更せず、NNUE重みを差し替えることで局面適応を実現
