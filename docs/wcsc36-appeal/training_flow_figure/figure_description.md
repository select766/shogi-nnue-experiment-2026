## 脚注

1. 「※ DNN Backbone には誤差を逆伝播しない (重み固定)。」

---

## 意味的に重要な設計上のポイント

1. **ルート局面 A のみが DNN に入力される**: ランダム局面 B は NNUE推論にのみ渡される。これにより「ルート局面の情報から、探索先の局面を正しく評価できる」モデルを学習する
2. **損失はランダム局面 B への評価値で計算**: ルート局面 A の評価値ではない。これがペア形式学習の核心
3. **誤差逆伝播の対象**: DNN Adapter と NNUE Experts の重みのみ更新。DNN Backbone は固定
4. **合成NNUE評価関数は「局面 A のルート専用」**: 推論時と同様に、1手番ごとに生成された評価関数を探索全体で使い続ける設定を反映
