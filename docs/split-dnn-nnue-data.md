現在、 `scripts/run_train_expert_blending_8experts_v4_paired_uniform50_noise0.sh` から呼ばれる学習コードでは、最初にDNN用とNNUE用がペアになった学習データのうちNNUE用の部分を別のファイルに分離する処理を行い、それからそれぞれのファイルを読んで学習を進めています。これは、NNUE用のデータが2倍のディスク容量を占めており非効率です。
これを解決するため、学習データを作成した直後に、分離スクリプトを実行し、最初からDNN用とNNUE用を別のファイルにしたいと思います。
例えば現在train.bin を学習データとして指定しているとして、train/{dnn.bin,nnue.bin}に予め分離するスクリプトを実行し、学習スクリプトにはtrainディレクトリを指定するようにしたいです。
学習データの作成（シャッフル）は scripts/run_paired_shuffle.sh で現在行っているので、これの最後に分離用のスクリプトを呼び出すことを予定します。
現在の、ファイル1つ(train.bin)を学習スクリプトとの入力とする機構は消去します。互換性を保つためにオプションを増やさず、今回の新しい機構で置き換えます。
以下のものを実装してください
- 分離スクリプト
- シャッフルスクリプトから分離スクリプトを呼ぶ処理
- 学習スクリプトから分離されたデータを読めるようにする変更
動作テストは、 `dataset/split_v1_paired_uniform_50` 以下にあるデータを少量(10kサンプル程度)切り取って実施してください。
