# エキスパートライブビジュアライザー 仕様書

## 目的

Expert Blending 対応やねうら王が対局中に、どのエキスパートをどの割合で使用しているかを  
観戦者向けにリアルタイム表示する。対象は AI に詳しくない一般観戦者であり、  
「この将棋 AI が局面の性質を自動で判断して戦略を切り替えている」という特徴を  
わかりやすく伝えることが目的。

---

## システム概要

```
YaneuraOu (対局中)
  │ stdout
  │  info string blending_weight=[0.123, 0.456, ...]
  ▼
tee → ログファイル
  ▼
Python サーバー (server.py)
  │ ログを末尾から読み、最新の blending_weight を返す
  │ HTTP polling または WebSocket
  ▼
ブラウザ (index.html)
  └── 右 1/3 パネルにリアルタイム表示
```

### YaneuraOu の stdout フォーマット

探索開始ごとに以下の行が出力される：

```
info string blending_weight=[0.123, 0.456, 0.789, 0.012, 0.034, 0.045, 0.067, 0.089]
```

- 8 個の float 値、合計 1.0（softmax 後）
- インデックスはエキスパート番号（0–7）に対応

### ヘッダー

- タイトル：**「局面適応型AIの専門家切替モニター」**
- 説明文：**「このAIは将棋の局面の特徴（序盤・終盤・戦型など）を自動で読み取り、  
  8人の"専門家（エキスパート）"の判断を組み合わせて最善手を探しています。  
  各バーの大きさが現在の専門家の貢献度です。」**

### エキスパートボックス（8個）

各ボックスの表示内容：

| 要素 | 内容 | 更新 |
|---|---|---|
| エキスパート番号 | `Expert 0` のような識別子 | 静的 |
| ラベル名 | `相振り飛車・終盤` などの日本語ラベル | 静的 |
| 手数帯の説明 | `主な局面: 100〜150手台` | 静的 |
| 代表局面画像 | 代表的な盤面 SVG 1枚 | 静的 |
| 混合比率（数値） | `45%` のような大きめ文字 | **動的** |
| 混合比率（背景色） | 0% → `#ffffff`、100% → `#ff7777` | **動的** |

### 色の仕様

```
ratio ∈ [0.0, 1.0]
R = 255
G = round(255 - 136 × ratio)   // 255 → 119
B = round(255 - 136 × ratio)
```

## サーバー仕様（`server.py`）

### エンドポイント

- `GET /` → `index.html` を返す
- `GET /api/weights` → 最新の混合比率 JSON を返す
  ```json
  {
    "weights": [0.05, 0.08, 0.12, 0.25, 0.03, 0.22, 0.10, 0.15],
    "timestamp": 1714000000.0,
    "ply": null
  }
  ```
- `GET /static/<path>` → SVG 画像などの静的ファイル配信

### ログ読み取り

- ログファイルパスはコマンドライン引数 `--log` で指定
- ファイル末尾から `info string blending_weight=[...]` を正規表現で検索し最新値を返す
- ファイルが存在しない・該当行がない場合は weights に `null` を返す

### ポーリング

ブラウザ側は 1 秒ごとに `/api/weights` をポーリングする（WebSocket は将来対応）。

## 起動方法（予定）

```bash
# YaneuraOuの出力をteeでファイルに保存しながら対局
./YaneuraOu ... 2>&1 | tee /tmp/yaneuraou.log

# 別ターミナルでビジュアライザーを起動
cd tools/expert_live_visualizer
python server.py --log /tmp/yaneuraou.log --port 8765

# ブラウザで http://localhost:8765 を開く（右 1/3 に配置）
```
